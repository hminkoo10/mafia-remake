# 마피아 레이팅 시스템

## 반영 조건

- 정상 승리 조건으로 끝난 게임만 반영합니다.
- 관리자 중지, 오류 정리, 모집 취소는 레이팅을 바꾸지 않습니다.
- `RunningGame.stats_recorded`로 한 게임을 한 번만 기록합니다.
- 통계는 Discord user id 기준으로 저장합니다.

## 기본 Elo

초기 레이팅은 `1000`입니다. 각 플레이어는 최종 진영 기준 상대 진영 평균 레이팅과 비교합니다.

```text
expected = 1 / (1 + 10 ^ ((opponent_average - rating) / 400))
team_delta = round(K * (score - expected) * player_count_multiplier)
```

| 조건 | K |
| --- | ---: |
| 레이팅 반영 10판 미만 | 40 |
| 10판 이상 30판 미만 | 32 |
| 30판 이상 | 24 |

| 참가 인원 | 배율 |
| --- | ---: |
| 3명 이하 | 0.60 |
| 4~6명 | 0.85 |
| 7~10명 | 1.00 |
| 11명 이상 | 1.10 |

승리 점수는 `1.0`, 패배 점수는 `0.0`입니다. 조커와 교주팀은 실제 최종 승자/최종 진영 기준으로 계산합니다.

## 역할 보정

게임 중 성공이 확정된 행동을 `rating_events`에 누적하고, 종료 때 역할 보정으로 합산합니다.

```text
role_delta = clamp(sum(success_events) + inactivity_penalty, -10, +10)
final_delta = clamp(team_delta + role_delta, -50, +50)
```

패배팀은 역할 활약이 있어도 최종 상승이 `+3`을 넘지 않습니다. 레이팅은 `0` 아래로 내려가지 않습니다.

핵심 능력이 있는 역할은 2일차 이후까지 생존했고 유효 행동을 한 번도 제출하지 않은 경우 `-2`를 받습니다. 사망자나 행동 기회가 불확실한 플레이어에게는 이 감점을 적용하지 않습니다.

## 저장과 로그

`stats.json` 사용자 항목은 현재 레이팅, 최고 레이팅, 반영 판수, 최근 20개 `rating_history`를 저장합니다.

각 기록은 다음 정보를 남깁니다.

- `team_delta`: 승패 Elo 변동
- `role_delta`: 역할 성공·미사용 보정
- `delta`: 최종 변동
- `rating_reasons`: 계산 근거
- 역할, 최종 진영, 승자, 참가 인원, 종료 시각

기존 전적의 승패·역할 횟수·플레이 시간은 레이팅과 별도로 계속 누적합니다.

## 확인 항목

- 성공 역할 이벤트가 `role_delta`와 레이팅 로그에 남는지
- 역할 보정이 `-10~+10` 범위를 넘지 않는지
- 핵심 능력 미사용 생존 특수직이 `-2`를 받는지
- 패배팀이 역할 보정만으로 `+3`을 넘지 않는지
- 게임 종료가 중복 기록되지 않는지
