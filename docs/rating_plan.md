# 마피아 레이팅 README

이 문서는 마피아 봇의 전적/레이팅 시스템을 운영자와 개발자가 같은 기준으로 이해하도록 정리한 문서입니다.

레이팅 코드는 주로 `src/stats.rs`에 있고, 게임 종료 시 `src/runner.rs`의 `announce_winner`에서 호출됩니다. 역할별 성공 이벤트는 `src/game/actions.rs`, `src/game/resolve.rs`에서 누적됩니다.

---

## 목표

레이팅 시스템의 목적은 단순 승률이 아니라 다음 요소를 함께 반영하는 것입니다.

- 진영 승리 여부
- 상대 진영 평균 레이팅 대비 기대 승률
- 참가 인원에 따른 게임 난이도
- 플레이어의 레이팅 구간
- 해당 판에서 맡은 역할의 실제 기여
- 핵심 능력 미사용 패널티

최근 개편 방향은 아래와 같습니다.

- 낮은 레이팅 플레이어가 점수를 올리기 쉬워야 합니다.
- 낮은 레이팅 플레이어는 패배해도 덜 떨어져야 합니다.
- 높은 레이팅 플레이어는 승리 보상이 줄어들어야 합니다.
- 높은 레이팅 플레이어는 패배 손실이 더 커져야 합니다.
- 한 판의 점수 변동 폭은 이전보다 커야 합니다.
- 레이팅 구간을 `C`, `B`, `A`, `S`, `SS`, `X` 랭크로 보여줘야 합니다.

---

## 반영 시점

레이팅은 정상적으로 승자가 결정된 게임 종료 시 한 번만 기록됩니다.

흐름:

1. 게임 루프가 `game.winner()`로 승자를 확인합니다.
2. 승자가 있으면 `game.phase = Phase::Ended`로 바꿉니다.
3. `RunningGame.stats_recorded`가 `false`인 경우만 전적을 기록합니다.
4. `stats::record_game_stats(...)`로 모든 참가자의 전적과 레이팅을 계산합니다.
5. `stats.json`에 저장합니다.
6. 게임 채널에 이번 판 레이팅 로그를 출력합니다.

중복 기록 방지:

- `RunningGame.stats_recorded`가 한 게임의 통계 중복 저장을 막습니다.
- 봇이 같은 종료 루틴을 여러 번 타도 한 번만 반영됩니다.

반영되지 않는 경우:

- 모집 취소
- 관리자 중지
- 수동 cleanup
- 승자가 없는 비정상 종료
- 통계 기록 전에 종료 루프가 정상 승자를 얻지 못한 경우

## 저장 구조

전적은 Discord user id 문자열을 key로 저장합니다.

주요 필드:

| 필드 | 의미 |
| --- | --- |
| `name` | 마지막으로 기록된 Discord 표시 이름 |
| `games` | 전체 게임 수 |
| `wins` | 승리 수 |
| `losses` | 패배 수 |
| `mafia_team_games` | 마피아팀으로 플레이한 횟수 |
| `play_seconds` | 누적 플레이 시간 |
| `rating` | 현재 레이팅 |
| `rating_games` | 레이팅 반영 판수 |
| `rating_peak` | 최고 레이팅 |
| `rating_history` | 최근 레이팅 변화 기록 |
| `roles` | 역할별 플레이 횟수 |

`rating_history`는 최근 20개만 유지됩니다.

`rating_history` 항목:

| 필드 | 의미 |
| --- | --- |
| `ended_at` | 게임 종료 시각 |
| `before` | 시작 전 레이팅 |
| `after` | 반영 후 레이팅 |
| `delta` | 최종 변화량 |
| `team_delta` | 승패 Elo 변화량 |
| `role_delta` | 역할 기여 보정 |
| `role` | 시작 시 역할 |
| `team` | 최종 레이팅 계산용 진영 |
| `winner` | 승리 진영 |
| `players` | 참가 인원 |
| `rating_reasons` | 계산 근거 텍스트 |

역할은 `initial_roles` 기준으로 기록합니다. 도둑, 도굴꾼, 광신도 등으로 게임 중 역할/진영이 바뀌어도 역할 플레이 횟수와 로그 표시는 시작 역할을 보존합니다.

## 초기값과 상한

| 항목 | 값 |
| --- | ---: |
| 초기 레이팅 | `1000` |
| 한 판 전체 변동 상한 | `-80 ~ +80` |
| 역할 보정 상한 | `-14 ~ +14` |
| 패배팀 최종 상승 상한 | `+5` |
| 첫 사망 패배 완화 | 최종 손실의 `25%` 완화 |
| 레이팅 최저값 | `0` |
| 레이팅 히스토리 보관 수 | `20` |

패배팀 최종 상승 상한은 역할 활약으로 패배했는데도 너무 많이 오르는 상황을 막기 위한 장치입니다. 예를 들어 패배팀이 `team_delta = -2`, `role_delta = +14`를 얻으면 합산은 `+12`지만 최종 상승은 `+5`까지만 허용됩니다.

첫 사망 패배 완화는 가장 먼저 사망한 플레이어가 그 판에서 패배했을 때만 적용됩니다. 최종 변화량이 음수일 경우 손실의 약 25%를 되돌려, 초반 사망으로 게임 영향력이 크게 줄어든 상황을 조금 보정합니다.

## 전체 계산 흐름

한 플레이어의 최종 변화량은 아래 순서로 계산됩니다.

```text
old_rating = 현재 레이팅
won = 플레이어가 승리 진영인지 여부
score = won이면 1.0, 패배면 0.0

opponent_average = 상대 진영 평균 레이팅
expected = 1 / (1 + 10 ^ ((opponent_average - old_rating) / 400))

base_delta = K * (score - expected)
team_delta = round(base_delta * rating_progression_multiplier * player_count_multiplier)
team_delta = clamp(team_delta, -80, +80)

role_delta = clamp(역할 이벤트 합계 + 미사용 패널티, -14, +14)
combined_delta = clamp(team_delta + role_delta, -80, +80)

if won:
    final_delta = combined_delta
else:
    final_delta = min(combined_delta, +5)

if first_dead and not won and final_delta < 0:
    final_delta += ceil(abs(final_delta) / 4)

after = max(old_rating + final_delta, 0)
```

최종 저장값:

```text
rating = after
rating_peak = max(rating_peak, after)
rating_games += 1
```

## 기대 승률 계산

기대 승률은 Elo 방식입니다.

```text
expected = 1 / (1 + 10 ^ ((opponent_average - player_rating) / 400))
```

의미:

- 내 레이팅이 상대 평균보다 높으면 기대 승률이 높습니다.
- 기대 승률이 높은 사람이 이기면 적게 오릅니다.
- 기대 승률이 높은 사람이 지면 크게 떨어집니다.
- 기대 승률이 낮은 사람이 이기면 크게 오릅니다.
- 기대 승률이 낮은 사람이 지면 적게 떨어집니다.

예시:

```text
내 레이팅 1000, 상대 평균 1000
expected = 0.5

승리 score = 1.0
score - expected = +0.5

패배 score = 0.0
score - expected = -0.5
```

## 상대 진영 평균 레이팅

상대 평균은 최종 레이팅 계산용 진영을 기준으로 고릅니다.

진영 key:

| 조건 | key |
| --- | --- |
| 조커 | `joker` |
| 교주팀 | `cult` |
| 마피아팀 | `mafia` |
| 그 외 시민팀 | `citizen` |

상대 후보 선택:

| 내 진영 | 상대 후보 |
| --- | --- |
| 시민팀 | 마피아팀 |
| 마피아팀 | 시민팀 |
| 교주팀 | 교주팀이 아닌 모든 플레이어 |
| 조커 | 조커가 아닌 모든 플레이어 |

상대 후보가 비어 있으면 자기 자신을 제외한 모든 플레이어를 후보로 사용합니다. 그래도 비어 있으면 자신의 현재 레이팅을 상대 평균으로 사용합니다.

---

## K 값

K 값은 레이팅 반영 판수에 따라 달라집니다. 초반에는 빠르게 자기 위치를 찾아가고, 게임 수가 쌓이면 변동이 완만해집니다.

| 레이팅 반영 판수 | K |
| --- | ---: |
| 10판 미만 | `56` |
| 10판 이상 30판 미만 | `48` |
| 30판 이상 70판 미만 | `40` |
| 70판 이상 | `34` |

의미:

- 신규/복귀 플레이어는 점수 이동이 큽니다.
- 오래 플레이한 사람은 한 판으로 순위가 과하게 흔들리지 않습니다.
- 이전 개편보다 K 값을 낮춰 승급 속도가 너무 빨라지는 문제를 줄였습니다.

## 레이팅 구간 보정

레이팅 구간 보정은 이번 개편의 핵심입니다.

승리 시:

| 현재 레이팅 | 승리 배율 |
| --- | ---: |
| 950 미만 | `1.45` |
| 950 이상 1125 미만 | `1.15` |
| 1125 이상 1325 미만 | `0.95` |
| 1325 이상 1575 미만 | `0.78` |
| 1575 이상 1900 미만 | `0.62` |
| 1900 이상 | `0.48` |

패배 시:

| 현재 레이팅 | 패배 배율 |
| --- | ---: |
| 950 미만 | `0.60` |
| 950 이상 1125 미만 | `0.78` |
| 1125 이상 1325 미만 | `0.95` |
| 1325 이상 1575 미만 | `1.12` |
| 1575 이상 1900 미만 | `1.32` |
| 1900 이상 | `1.55` |

효과:

- 하위권은 이기면 크게 오르고, 져도 덜 떨어집니다.
- 중간권은 비교적 표준 Elo에 가깝게 움직입니다.
- 상위권은 이겨도 조금씩 오르고, 지면 크게 떨어집니다.

레이팅 로그에는 배율이 1.0이 아닌 경우 `레이팅 구간 보정 x1.15` 같은 사유가 남습니다.

## 참가 인원 보정

게임 규모에 따라 변동폭을 조절합니다.

| 참가 인원 | 배율 |
| --- | ---: |
| 3명 이하 | `0.60` |
| 4~6명 | `0.85` |
| 7~10명 | `1.00` |
| 11명 이상 | `1.10` |

의미:

- 소규모 게임은 변수와 역할 분포가 좁아서 변동폭을 줄입니다.
- 일반 규모 게임은 기본값을 씁니다.
- 대규모 게임은 승리 난이도와 기여 난이도가 높아서 조금 더 반영합니다.

## 역할 보정

역할 보정은 승패와 별개로 게임 중 실제 행동을 점수화합니다.

```text
role_delta = clamp(이벤트 점수 합계 + 미사용 패널티, -14, +14)
```

역할 보정 특징:

- 한 판 역할 보정은 최대 `+14`, 최소 `-14`입니다.
- 역할 보정은 승패 Elo와 합산됩니다.
- 패배팀은 역할 보정으로 최종 `+5`까지만 오를 수 있습니다.
- 첫 사망자가 패배하면 최종 손실의 25%가 완화됩니다.
- 성공 이벤트가 여러 번 있어도 상한 이상은 잘립니다.
- 상한에 걸리면 로그에 `직업 보정 상한 적용`이 남습니다.

### 역할 이벤트가 생기는 방식

게임 중 아래 두 가지가 기록됩니다.

| 데이터 | 의미 |
| --- | --- |
| `rating_action_counts` | 핵심 능력을 제출했는지 여부 |
| `rating_events` | 실제 성공/기여 이벤트 |

능력 제출만으로도 미사용 패널티는 피할 수 있습니다. 하지만 점수 상승은 별도 성공 이벤트가 있어야 합니다.

예시:

- 경찰이 조사 대상을 제출했지만 마피아팀을 찾지 못함: 미사용 패널티 없음, 성공 점수 없음
- 경찰이 조사로 마피아팀을 찾음: `+4`
- 의사가 치료 대상을 제출했지만 공격을 막지 못함: 미사용 패널티 없음, 성공 점수 없음
- 의사가 마피아 공격을 막음: `+5`

- 최종 반영은 `ROLE_DELTA_CAP = 14`로 제한됩니다.

## 핵심 능력 미사용 패널티

핵심 능력이 있는 역할이 능력을 전혀 제출하지 않고 2일차 이후까지 생존하면 `-2`를 받습니다.

조건:

```text
action_count == 0
player.alive == true
game.day_number >= 2
role_has_core_action(role) == true
```

패널티 이유:

- 살아 있었고 행동 기회가 있었는데 핵심 능력을 쓰지 않은 경우를 약하게 감점합니다.
- 사망자에게는 적용하지 않습니다.
- 1일차 이전 짧은 게임에는 적용하지 않습니다.
- 능력을 제출했지만 결과가 없던 경우에는 적용하지 않습니다.

미사용 패널티 대상 역할:

| 진영 | 역할 |
| --- | --- |
| 시민팀 | 의사, 간호사, 건달, 경찰, 자경단원, 기자, 해커, 심리학자, 최면술사, 용병, 사립탐정, 영매, 성직자 |

| 교주팀 | 교주, 광신도 |

코드상 대상:


## 최종 진영과 승리 판정

플레이어가 이겼는지는 `player_won_game`으로 판단합니다.

| 승자 | 이기는 플레이어 |
| --- | --- |
| 마피아 | 최종 마피아팀 |
| 시민 | 최종 시민팀 |
| 교주팀 | 최종 교주팀 |
| 조커 | 조커 승리자 |

마피아팀/교주팀 여부는 게임 상태를 기준으로 봅니다. 접선, 포교, 재림 등으로 최종 진영이 바뀌면 레이팅 승패 판정에도 영향을 줄 수 있습니다.

조커는 단독 승리 구조입니다. `joker_winner_id`가 있으면 해당 user id만 조커 승리자로 처리합니다.

## 랭크 제도

랭크는 현재 레이팅을 사람이 보기 쉽게 나눈 표시입니다. 레이팅 계산에는 직접 영향을 주지 않습니다.

| 레이팅 | 랭크 |
| ---: | --- |
| 950 미만 | `C` |
| 950 이상 1100 미만 | `B` |
| 1100 이상 1300 미만 | `A` |
| 1300 이상 1550 미만 | `S` |
| 1550 이상 1850 미만 | `SS` |
| 1850 이상 | `X` |

초기 레이팅 `1000`은 `B` 랭크입니다.

랭크 표시 위치:

- Discord `/내정보`
- Discord `/리더보드` 이미지
- 웹 `/leaderboard`
- 웹 홈 TOP 3
- 공개 API `/api/leaderboard`
- 보호 API `/api/v1/leaderboard`

API 필드:

| 필드 | 의미 |
| --- | --- |
| `rating` | 현재 레이팅 |
| `rating_rank` | 현재 랭크 |
| `rating_peak` | 최고 레이팅 |
| `rating_peak_rank` | 최고 레이팅 기준 랭크 |
| `rating_games` | 반영 판수 |

## 출력 예시

게임 종료 후 채널에 "이번 판 레이팅 로그"가 출력됩니다.

형식:

```text
- 닉네임 (역할) 1000 -> 1042 (+42) [팀 +37 / 직업 +5]
  사유: 소속 진영 승리, 마피아 공격 치료 성공 +5, 레이팅 구간 보정 x1.35
```

의미:

| `1000 -> 1042` | 이전/이후 레이팅 |
| `+42` | 최종 변화량 |
| `팀 +37` | 승패 Elo 변화 |

## 계산 예시

### 예시 1. 1000점 플레이어가 1000점 평균 상대에게 승리

조건:

```text
old_rating = 1000
opponent_average = 1000
rating_games = 0
players = 8
won = true
role_delta = +5
```

계산:

```text
expected = 0.5
K = 56
rating_progression_multiplier = 1.15
player_count_multiplier = 1.0

base_delta = 56 * (1.0 - 0.5) = 28
team_delta = round(28 * 1.15 * 1.0) = 32
combined_delta = clamp(32 + 5, -80, +80) = 37
final_delta = 37
after = 1037
```

결과:

```text
1000 -> 1037 (+37)
```

### 예시 2. 850점 플레이어가 패배

조건:

```text
old_rating = 850
opponent_average = 1000
rating_games = 5
players = 8
won = false
role_delta = 0
```

효과:

- 낮은 레이팅이라 패배 배율 `0.55`가 적용됩니다.
- 기대 승률도 낮으므로 패배 손실이 더 작습니다.

결과적으로 낮은 구간 플레이어는 패배해도 크게 무너지지 않습니다.

### 예시 3. 1750점 플레이어가 패배

조건:

```text
old_rating = 1750
opponent_average = 1300
rating_games = 80
players = 8
won = false
role_delta = 0
```

효과:

- 높은 레이팅이라 패배 배율 `1.55`가 적용됩니다.
- 상대 평균보다 훨씬 높기 때문에 기대 승률이 높습니다.
- 기대 승률이 높은 상태에서 패배했으므로 손실이 커집니다.

상위권은 유지하려면 계속 안정적으로 이겨야 합니다.

### 예시 4. 패배했지만 역할 활약이 큰 경우

조건:

```text
team_delta = -2
role_delta = +14
won = false
```

계산:

```text
combined_delta = +12
final_delta = min(+12, +5) = +5
```

결과:

```text
패배팀 상승 제한 적용
최종 +5
```

패배팀도 잘하면 조금 오를 수 있지만, 승리팀보다 크게 오르지는 못합니다.

## 튜닝 가이드

레이팅을 더 빠르게 움직이고 싶을 때:

- `RATING_DELTA_CAP` 증가
- `rating_k` 값 증가
- 승리 구간 보정 배율 증가
- 참가 인원 배율 증가

하위권을 더 보호하고 싶을 때:

- 950 미만 패배 배율을 더 낮춤
- 950~1125 패배 배율을 더 낮춤
- `LOSING_RATING_GAIN_CAP`은 유지하거나 소폭 증가

상위권을 더 빡세게 만들고 싶을 때:

- 1575 이상 승리 배율을 더 낮춤
- 1575 이상 패배 배율을 더 높임
- X 랭크 기준을 1850보다 높임

역할 기여도를 더 크게 보고 싶을 때:

- `ROLE_DELTA_CAP` 증가
- 개별 `record_rating_event` 점수 증가
- 성공 이벤트를 더 많이 추가

주의:

- `RATING_DELTA_CAP`만 키우고 K/배율을 그대로 두면 실제 변화는 크게 늘지 않을 수 있습니다.
- `ROLE_DELTA_CAP`을 너무 키우면 패배팀이 승패보다 역할 점수로 보상받는 느낌이 강해집니다.
- 패배팀 상승 제한을 너무 높이면 "졌는데 많이 오르는" 판이 자주 나옵니다.
- 랭크 경계값을 바꾸면 기존 플레이어들의 표시 랭크가 즉시 바뀝니다.

## 테스트 포인트

레이팅 변경 시 최소 확인할 항목:

- `rating_rank` 경계값이 맞는지
- 낮은 레이팅 승리 배율이 높은 레이팅 승리 배율보다 큰지
- 낮은 레이팅 패배 배율이 높은 레이팅 패배 배율보다 작은지
- 역할 보정이 `-14 ~ +14`를 넘지 않는지
- 전체 변동이 `-80 ~ +80`을 넘지 않는지
- 패배팀 최종 상승이 `+5`를 넘지 않는지
- 첫 사망 패배 완화가 첫 사망자에게만 적용되는지
- `rating_history`가 최근 20개만 남는지
- `/내정보`, `/리더보드`, 웹 리더보드, API에 랭크가 표시되는지

현재 관련 테스트:

- `rating_rank_maps_rating_bands`
- `rating_progression_helps_lower_ratings_more`
- `role_rating_adjustment_is_capped`
- `losing_team_positive_gain_is_capped`
- `first_dead_losing_player_loses_less_rating`
- `leaderboard_sorts_by_rating`
- `successful_role_event_is_recorded_in_rating_history`
